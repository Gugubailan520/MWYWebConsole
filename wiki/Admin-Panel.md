# 管理后台

管理后台提供运行期可视化配置，修改后立即生效，无需重启服务。

## 访问方式

1. 使用 `config.yml` 中配置的 admin 账号登录主站
2. 顶栏右侧出现「管理」按钮，点击进入
3. 或直接访问 `/admin.html`（需先登录 admin 账号）

> 管理员账户仅在 `config.yml` 中配置，不入数据库，无法在后台修改。详见 [配置参考](Configuration)。

---

## 功能模块

### 1. 基本设置

自定义站点外观，所有设置即时生效：

| 设置项 | 说明 | 示例 |
|--------|------|------|
| 站点名称 | 顶栏品牌名和浏览器标题 | `冥雾云Web控制台` |
| 副标题 | 站点名称下方的灰色小字 | `MWY Web Console` |
| Logo URL | 顶栏 Logo 图片地址，支持本地上传或外部 URL | `/uploads/logo.png` |
| Favicon URL | 浏览器标签页图标 | `/uploads/favicon.ico` |
| 背景图 URL | 登录页/主页背景图片 | `/uploads/bg.png` |
| 主题色 | 强调色（按钮、选中态等），支持颜色选择器 | `#3a86ff` |
| 页脚文本 | 页面底部显示的文字，留空不显示 | `© 2025 冥雾云` |

> 图片类设置支持直接输入 URL，也可通过文件上传功能上传到 `data/uploads/` 目录。

### 2. 功能开关

按需开关各协议和系统功能：

| 开关 | 默认 | 说明 |
|------|------|------|
| SSH 终端 | 启用 | 关闭后主页「添加服务器」弹窗中不显示 SSH 选项 |
| SFTP 文件管理 | 启用 | 关闭后不显示 SFTP 选项 |
| RDP 远程桌面 | 启用 | 关闭后不显示 RDP 选项 |
| VNC 远程桌面 | 启用 | 关闭后不显示 VNC 选项 |
| FTP 文件传输 | 启用 | 关闭后不显示 FTP 选项 |
| 用户注册 | 启用 | 关闭后注册入口隐藏，仅管理员可创建用户 |

> 关闭某协议后，已存在的该类型服务器卡片仍可见，但无法新建该类型的连接。

### 3. 代理设置

配置全局 SOCKS5 出站代理，所有协议连接可经代理转发：

| 配置项 | 说明 |
|--------|------|
| 启用代理 | 总开关，关闭后所有连接直连 |
| 代理地址 | SOCKS5 代理服务器地址 |
| 端口 | 代理端口 |
| 用户名 | 代理认证用户名（可选） |
| 密码 | 代理认证密码（可选） |
| 应用到 SSH | SSH/SFTP 连接走代理 |
| 应用到 RDP | 暂不支持（RDP 走 guacd 独立通道） |
| 应用到 VNC | VNC 连接走代理 |
| 应用到 FTP | FTP 连接走代理 |

> 代理连接失败时自动回退到直连，不会阻断用户操作。

### 4. 用户管理

管理普通用户账号（管理员账号在 `config.yml` 中配置）：

- **创建用户** — 设置用户名和初始密码
- **禁用/启用** — 暂停账号访问权限
- **重置密码** — 为用户设置新密码
- **删除用户** — 永久删除账号及其数据

### 5. 连接日志

查看所有连接记录，用于审计和统计：

- 记录字段：用户、协议类型、目标地址、端口、连接时间、持续时间
- 支持按协议类型筛选
- 支持导出为 CSV 文件
- 根据 `config.yml` 中的 `log_retention_days` 自动清理过期记录

## 技术细节

### 样式注入机制

`site-style.js` 在所有页面的 `<head>` 中同步加载，启动时拉取 `/api/site/info` 接口获取站点设置，然后：

- 替换 `document.title` 和 `<link rel="icon">`
- 替换 `body` 背景图、Logo `img` 的 `src`、品牌名文本
- 设置 CSS 变量（`--accent`、`--accent-glow` 等）

管理后台页面（`admin.html`）设置 `window.__ADMIN_PAGE = true`，`site-style.js` 检测到此标记后仅应用标题和 favicon，不覆盖后台的视觉样式。

### API 鉴权

管理后台 API（`/api/admin/*`）使用 JWT 中间件 `requireAdmin` 验证：

1. 前端登录后 token 存入 `sessionStorage.admin_token`
2. 每次 API 请求携带 `Authorization: Bearer <token>` 头
3. 中间件验证 token 签名和用户角色

### 数据存储

- 启动期配置（端口、密钥、管理员）→ `config.yml`
- 运行期数据（站点设置、用户、日志）→ SQLite (`data/console.db`)
- 上传资源（Logo、背景图等）→ `data/uploads/`
